함수 attach는 사용할 data frame을 기본적으로 연결해 놓는 함수로 이해하면 된다. 더 확장된 의미로 사용자가 기본적으로 사용할 position (서랍장…)을 지정하는 기능이다.

위의 표현은 변수가 많은데 해당되는 변수 이름의 값들만 추출하고 싶을 때 유용하다.

함수 dettach

attach된 상태를 원상복귀하려면 detach하여야 한다.

detach()

오류가 발생한 것은 detach 되었기 때문이다.

Make your own function

  • R을 하용하는데 있어 가장 중요한 것은 R에 제공하는 함수만을 활용하는 것뿐 아니라, 본인의 함수를 직접 만드는 것이다.

  • 함수를 만들기 위해서는 프로그램 능력이 요구되며 이 능력은 많은 시행착오와 고민에 의해 달성될 수 있다.

*R의 함수 작성을 위한 기본골격은 간단하다.

  • 기본골격은 function name <- function(arguments){ program }
    이다.

  • 예를 들면, 

my.mean <- function(foo) {
    tt <- sum(foo)/length(foo)
    return(tt)
}
  • 함수이름은 my.mean 이고 foo 가 argument로 numeric vector가 요구된다.
  • 그리고 두 줄의 프로그램을 통해 foo의 평균이 tt 에 저장되어 우리에게 제공된다.
my.mean(1:10)

## [1] 5.5

mean(1:10)

## [1] 5.5
  • 여기서 주의할 것은 return(tt) 이다. 만약 return을 하지 않으면 다음과 같은 현상이 발생한다. 
my.mean <- function(foo) {
    tt <- sum(foo)/length(foo)
}
my.mean(1:10)
  • 즉, 아무 것도 얻는 값이 없다. 이것은 함수에서 계산한 값이 tt에 임시저장된 후 사라지기 때문이다.
  • 좀 더 자세히 이야기하자면, 함수에서 사용된 tt 와 같은 모든 메모리는 함수 작업이 끝느면서 사라지는 것이다.
  • 함수 내에서 계산된 결과를 우리가 받으려면 return 을 통해 반드시 전달 받아야 하는 것이다.
  • 이와 같이 함수는 작업 후 모든 메모리를 빈 상태로 만들어 주므로, 한정된 컴퓨터의 메모리를 활용하는데 필수적인 방법인 것이다.
